# Style 設計書

本ドキュメントは、Burn Your Own Style システムの**設計思想・基本コンセプト・変数設計**を定義する。

各 CustomClass（Cards, Panel, ImgText など）の詳細な使用方法については [`CLASS.md`](./CLASS.md) を参照。

---

## 設計思想と基本コンセプト

エージェントはユーザーが明示的に許可する時以外は**scssファイルを編集しない。**

### レスポンシブ設計方針

- **CustomClass**: DesktopFirst レスポンシブ対応（`@max-md` 等を使用）
- **Tailwind CSS**: デフォルトの MobileFirst（`md:`, `sm:` 等）と併用可能

### Tailwind CSS のバージョン

本プロジェクトでは **Tailwind CSS v3 のみ**を使用する。**v4 系**（例: `@tailwindcss/postcss`）は採用しない。

### コンテンツ幅の制御原理

**「基本コンテンツ幅を `--wid` で設定し、画面幅がそれ以上の時は中央配置」**

```scss
main>* {
    width: var(--wid);  // 固定幅（小さい画面でも--wid を維持）
    max-width: 100%;    // 画面より広くなったら縮小
    margin-inline: auto; // 中央配置
}
```

### 左右余白の計算原理

**`--into` と `--out` の関係**：

コンテンツを中央に配置し、`--wid` の幅を維持するための計算式は以下のように設計されている：

```scss
/* 左右のマージン（コンテンツ幅を画面幅にする） */
--out: calc(50% - 50vw);
/* 左右のパディング（中身が--wid の幅になる） */
--into: calc(50vw - clamp(0px, 50vw - var(--PX), (var(--wid) / 2)));
```

**`--into` の計算式の読み解き**：

この式は「画面幅が `--wid` より広い時にのみ、中身が`--wid`の幅になるようにpaddingをつける。

**`--out` の計算式**：

```scss
--out: calc(50% - 50vw);
```

これは「コンテンツ幅が画面幅になるように」計算する式。`50%` から `50vw` を引くことで、画面幅がコンテンツ幅よりも広くなった時のみ、その差分をマージンとして扱う。

**フォールバック値について**：

変数使用時は `var(--PX, 20px)` のようなフォールバック値は含めず、シンプルに `var(--PX)` と記載します。フォールバック値を指定すると：

- 変数が未設定の場合、フォールバック値が適用されるため「変数が定義されていない」という問題がわかりにくくなる
- 記述量が増え、可読性が低下する

そのため、**すべての変数は必ず `_01variables.scss`またはクラス内で定義することを前提**とし、フォールバック値は省略する。

## 変数設計の基礎

### フォントサイズの変数設計

本システムでは、**基本フォントサイズは `rem` 単位で指定**し、タイトル（`h1`, `h2`, `h3` 等）のみ流動的な値（`clamp()`）を使用する。

```scss
:root {
    --FZ: 1rem;
    @include max-md {
        --FZ: 0.875rem;
    }
    --h1FZ: clamp(24px, 4.8vw, 48px);
    --h2FZ: clamp(20px, 3.2vw, 32px);
    --h3FZ: clamp(18px, 2.4vw, 24px);
}
```

### 色変数の設計

**すべての色は変数で設定する**：

```scss
:root {
    --MC: #2db542;  // メインカラー
    --SC: #3194c9;  // サブカラー
    --AC: #512db5;  // アクセントカラー
    --BC: #fbf9ef;  // ベースカラー
    --TC: #333;     // テキストカラー
    // ...
}
```

Tailwind の arbitrary value 構文を使用し、CSS 変数を直接使用する：

- **background-color**: `bg-[--MC]`（メインカラー）
- **text-color**: `text-[--TC]`（テキストカラー）
- **border-color**: `border-[--SC]`（サブカラー）

## 基本変数

### レイアウト変数

| 変数名 | 例 | 説明 | 使用シーン |
|--------|------------|------|-----------|
| `--gap` | `2rem` | flex コンテンツの基本ギャップ | Cards, Panel, FlexR  |
| `--wid` | `1080px` | `main>*` の width になる幅 | `main>*` |
| `--MY` | `4rem` | セクション間の基本縦余白 | `main>*+*`, `.wrapper>*+*` の margin-top |
| `--PX` | `1.5rem` | 全てのコンテンツの基本横余白 | `main` の padding-inline |
| `--into` | 計算値 | 中身が--wid の幅になるように横余白を指定 | `.intoL`, `.intoR` |
| `--out` | 計算値 | コンテンツ幅を画面幅にする | `.out`, `.outR`, `.outL` |
| `--imgW` | `30%` | 画像幅比率 | ImgText, Panel で使用 |
| `--itemW` | 計算値 | Cards アイテム幅 | `.Cards` クラスで使用 |
| `--few` | `30%` | FlexR の小さい要素幅（各 Flex クラスで既定値が入り、上書き可） | Flex46, Flex37 など |

### 派生変数

以下の派生変数は、基本変数から計算される値で、実際に使用されています：

| 変数名 | 計算式 | 説明 |
|--------|--------|------|
| `--PX2` | `calc(var(--PX) * 2)` | PX の 2 倍 |
| `--gapN` | `calc(var(--gap)*-1)` | ギャップの負の値（ネガティブマージン用） |
| `--gapH` | `calc(var(--gap)/2)` | ギャップの半分 |

### 色変数

| 変数名 | 値（例） | 説明 |
|--------|---------|------|
| `--MC` | `#2db542` | メインカラー |
| `--SC` | `#3194c9` | セカンダリーカラー |
| `--AC` | `#512db5` | アクセントカラー |
| `--BC` | `#fbf9ef` | ベースカラー |

## 基本的な構造とユーティリティ

### `main>*` と `.wrapper` の役割

**`main>*`（ページの主要コンテンツ）**：
- デフォルトで `--wid`（1080px）の幅を持つ
- 自動的に中央配置される
- 要素間の縦余白は `--MY` が適用される

**`.wrapper`（セクション用ラッパー）**：
- セクション全体を囲む時に使用
- 上下の padding を `--MY` で制御
- 左右の margin を `--out` で制御（コンテンツ幅を画面幅にする）
- 内部要素は自動的に中央配置される

**`.wid`, `.PX`, `.MY`（部分適用用ユーティリティ）**：
- `main>*` や `.wrapper` ではない要素に、部分的に変数を適用したい時に使用
- 例：`.PX` で左右パディングを指定するなど

**`.intoL`, `.intoR`, `.out`, `.outR`, `.outL`（左右個別制御用）**：
- `padding-inline` や `margin-inline` ではなく、左右別々に制御したい時に使用
- 例：`intoL` は左側のみのインナーパディング、`outR` は右側のみのアウトマージン

## _20utility.scss（@layer globals）

CustomClass の Util クラスを `@layer globals` で設定する。class コンテナにクラスを付与し、子（`img` タグ等）のプロパティを一括指定する。

**参照の正**: 本リポジトリではビルドに取り込まれる **`src/scss/` 配下**を見る。プロジェクト直下の **`scss/` は参照用**（別系統のソース・複製）であり、ファイル名や分割は一致しない場合がある。

```scss
@layer globals {
        .img16-9 :is(img,video){aspect-ratio: 16/9;}
        .img3-2 :is(img,video){aspect-ratio: 3/2;}
        ...
}
```
