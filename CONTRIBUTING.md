# コントリビュート手順

このプロジェクトは、feature ブランチと Pull Request を前提に進めます。

## 基本の流れ

1. `main` から feature ブランチを切ります。
2. 並行作業が必要なら、同じ feature 系ブランチのローカル worktree を分けます。
3. 1 ブランチにつき 1 つの作業に絞るのが基本です。
4. 変更は小さく、レビューしやすい単位で commit します。
5. 早めに remote へ push します。
6. `main` に向けて Pull Request を作成します。
7. レビューで指摘があれば修正して、同じブランチに push します。
8. 確認が終わったら PR を merge します。
9. merge 後は必要に応じてローカルの `main` を pull します。

## 運用メモ

- `main` は常に確認しやすい状態を保ちます。
- ブランチ名は `feature/<topic>` のように、内容がわかる名前にします。
- 生成物や一時ファイルは、必要な場合を除いて commit しません。
- このリポジトリでは、開発用 UI の表示は本番ビルドに出さない前提です。

## 確認

- PR を出す前に、変更内容に応じたテストを実行します。
- UI や bridge を触ったときは、可能ならブラウザでも確認します。
