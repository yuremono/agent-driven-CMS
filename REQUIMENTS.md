
1. プロジェクト概要
Codex app-serverをローカル環境に組み込み、ブラウザ上のチャットUIから自然言語でウェブサイトを編集・デプロイできるCMSテンプレート。
既存のウェブサイトにこのパッケージを追加インストールすることで、ChatGPT Plus/Proのサブスクリプションを持つユーザーが、ターミナルやコードエディタを操作せずにサイトの編集・改善を行えるようにする。

2. ゴール

自然言語の指示だけでサイトの編集・テスト・デプロイが完結する
クライアントが日常的に使う操作はブラウザUIのみで完結する
GitHubテンプレートとして公開し、誰でも自分のサイトに導入できる


3. 対象ユーザーと前提条件
対象ユーザー：

このテンプレートを自分のクライアントに導入したいウェブ制作者
自分のサイトを自分でAI編集したい開発者

前提条件：

macOS環境
ChatGPT Plus / Pro サブスクリプション（OpenAI APIキー方式で認証）
初回インストールは制作者が立ち会って実施することを想定


4. 機能要件
4.1 コア機能

https://developers.openai.com/codex/app-server
ブラウザ上にチャットウィンドウUIを表示し、自然言語で編集指示を入力できる
Codex app-server経由で以下を自律実行：

サイトのファイル構造・コードの読み取り
自然言語指示に基づくコード生成・修正
Vercel Agent Browserによる自動テスト・動作確認
Gitへの自動コミット
本番環境への自動デプロイ（Vercel / Netlify / Cloudflare対応）



指示例：「トップページの色を春らしく変えて」「予約フォームを追加してGoogleカレンダーと連携させて」「料金表を最新価格に更新してモバイル対応に」

4.2 ユーザー体験

通常操作はブラウザUIのみで完結（ターミナル不要）
Codexの処理過程をストリーミング表示
「デプロイしますか？」の最終確認フロー
変更履歴の可視化（Git diff表示）
app-serverの起動・終了をブラウザUIから制御できる（理想）

実現できない場合はターミナルコマンドとマニュアルで対応



4.3 テンプレートとしての機能

既存のNext.jsサイトへの追加インストールを想定したパッケージ構成
！理想はどんな技術スタックからでも導入可能(htmlからでもvueからでも)
.env.example にOpenAI APIキーなど最小限の設定項目のみ記載
トラブルシューティング用ターミナルコマンドをREADMEに明記


5. 技術構成
フロントエンド： Next.js（App Router）＋ React
！パフォーマンス、スケーラビリティ重視で変更可能性を検討
通信レイヤー： ブラウザ ↔ Next.js APIルート ↔ Codex app-server（stdio）のWebSocket接続
バックエンド： Codex app-server（codex app-serverコマンドをローカルで起動）
認証： OpenAI APIキー方式（account/login/startにAPIキーを渡す）
その他： GitHub連携、Playwright自動テスト、Vercel / Netlify / Cloudflare デプロイ対応

6. 制約・注意点

対象OSはmacOSのみ（Windows対応はMVPの品質次第）
ChatGPT Plus / Pro サブスクリプションが必要
app-serverはローカルで常駐起動が必要（作業中はバックグラウンドで動作）
初回インストールは制作者が立ち会って実施することを前提とする
公開テンプレートとして配布するため、権限管理と認証の実装は必須


7. 成功イメージ
