# Product Vision

このファイルは、実装の正解を固定するためではなく、ユーザーが求めている運用イメージを記録するためのものです。  
PoC のダッシュボードではなく、実際の公開ページに溶け込む編集用チャットを目指します。

## 目指す見た目

- コーポレートサイトのような、一般的な公開ページに見えること
- トップページに AI チャットが大きく置かれたサービスの印象に近いこと
- ただし、編集専用の UI として自然にサイトへ重なること
- 管理画面やデバッグ画面に見えすぎないこと

## チャットの配置

- チャットウィンドウは画面下端から少し内側に入った位置で固定表示する
- サイドバー常駐ではなく、全画面の印象を保ったまま重ねる
- チャットウィンドウ自体も小さく折りたためるとよい

## エージェント回答の扱い

- 送信後の回答は、チャットウィンドウの上に広がるエリアで表示する
- 回答エリアは、送信前には表示しない
- まずは余計な制御を減らし、必要になったら後から折りたたみを再検討する
- 理由は、編集対象のウェブサイト本体が見えなくならないようにするため

## 公開ページと編集 UI の分離

- ローカル環境では CMS 機能付きのプレビューを見せる
- 外部からアクセスする公開ページでは、チャットウィンドウを出さない
- つまり、公開サイトの見た目は保ちつつ、ローカル環境だけ編集 UI を載せる
- ルート `/` は公開トップ、`/services` は公開の複数ページ確認用、`/admin` は bridge 管理ビューとして扱う

## 実装の暫定方向

- まずはコーポレートサイト風の見た目を簡単にでも作る
- その上に編集用チャットを固定表示で載せる
- 編集 UI は development 時だけ全ページ共通で重ねる
- チャット UI は中央下寄せで、送信ボタン以外の操作を最小化する
